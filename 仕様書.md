# 仕様書: 商品デザイン確認アプリ

## 1. 技術スタック

| 項目 | 技術 |
|---|---|
| フロントエンド | Next.js (App Router) + TypeScript |
| UIフレームワーク | Tailwind CSS |
| バックエンド | Next.js API Routes |
| データベース | SQLite (Turso) または PostgreSQL (Vercel Postgres) |
| ファイルストレージ | Vercel Blob Storage |
| デプロイ | Vercel |

> **補足**: 利用規模が小さい（同時5名）ため、軽量な構成を採用。

---

## 2. 画面設計

### 2.1 画面一覧

| 画面名 | パス | デバイス | 説明 |
|---|---|---|---|
| トップ画面（検索） | `/` | スマホ/タブレット | 商品管理番号の検索 |
| 検索一覧画面 | `/search?q={検索文字列}` | スマホ/タブレット | 複数ヒット時のテキスト一覧 |
| 商品ページ | `/products/{親番号}` | スマホ/タブレット | デザイン一覧表示 |
| デザイン詳細 | `/products/{親番号}/{デザイン番号}` またはモーダル | スマホ/タブレット | 拡大画像 + コメント表示 |
| 管理画面トップ | `/admin` | PC | 管理機能メニュー |
| 画像アップロード | `/admin/upload` | PC | 画像一括アップロード |
| Excelアップロード | `/admin/excel` | PC | コメント一括登録 |
| 商品管理 | `/admin/products` | PC | 商品情報の編集・削除 |

### 2.2 トップ画面（検索画面）

#### レイアウト
```
┌──────────────────────────────┐
│                              │
│  [lk ]  [7] [8] [9]         │
│  [UV ]  [4] [5] [6]         │
│  [UVCHY] [1] [2] [3]        │
│  [cp ]      [0]             │
│                              │
│  [ 入力欄: lk027    ] [検索] │
│                              │
└──────────────────────────────┘
```

#### 操作仕様
- プレフィックスボタン押下 → 入力欄にプレフィックス文字列を挿入
- テンキー押下 → 入力欄に数字を追記
- 入力欄に直接キーボード入力も可能
- 「検索」ボタン押下、またはEnterキーで検索実行
- 入力欄クリア機能（バックスペースまたはクリアボタン）

### 2.3 検索一覧画面

#### 表示内容
- ヒットした親番号のテキスト一覧
- 各行タップで商品ページに遷移
- 「戻る」ボタンでトップ画面に戻る

#### 検索ロジック
- 入力文字列による前方一致検索
- 1件のみヒット → 一覧画面をスキップし、直接商品ページへ遷移
- 0件 → 「該当する商品が見つかりません」メッセージ表示

### 2.4 商品ページ

#### 表示内容
- 画面上部: 親番号の表示（例: `lk027`）
- デザインサムネイル一覧（グリッド表示）
  - 各サムネイルにデザイン番号ラベル（a, b, c...）を表示
- タップでデザイン詳細に遷移

### 2.5 デザイン詳細画面

#### 表示内容
- 拡大画像（ピンチイン・ピンチアウト対応）
- 管理番号表示（例: `lk027-a`）
- コメントテキスト表示
- 「戻る」ボタンでデザイン一覧に戻る
- 左右スワイプで前後のデザインに移動（任意）

## 3. 管理画面設計

### 3.1 画像アップロード画面

#### 操作フロー
1. 「ファイルを選択」ボタンから複数画像を選択（またはドラッグ&ドロップ）
2. 選択されたファイル名の一覧を表示
3. ファイル名バリデーション
   - 形式: `{プレフィックス}{3桁数字}-{a-y}.{jpg|png}`
   - 不正なファイル名はエラー表示し、スキップ可能
4. 「アップロード」ボタンで一括登録
5. 同一管理番号の画像が存在する場合は上書き
6. 結果サマリーを表示（成功件数、スキップ件数、エラー件数）

### 3.2 Excelアップロード画面

#### 操作フロー
1. `.xlsx` ファイルを選択
2. プレビュー表示（A列: 管理番号、B列: コメント）
3. バリデーション
   - A列の管理番号の形式チェック
   - 空行のスキップ
4. 「登録」ボタンで一括登録
5. 既存コメントがある場合は上書き
6. 結果サマリーを表示

### 3.3 商品管理画面

#### 機能
- 商品一覧表示（親番号でソート）
- 親番号で絞り込み検索
- 各商品の展開 → デザイン一覧表示
- 個別デザインの操作
  - 画像差し替え
  - コメント編集（インライン編集）
  - 削除（確認ダイアログ付き）
- 親番号単位の一括削除（確認ダイアログ付き）

## 4. データベース設計

### 4.1 テーブル定義

#### prefixes（プレフィックステーブル）
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー、自動採番 |
| name | TEXT | プレフィックス名（例: `lk`） |
| color | TEXT | ボタン表示色（例: `#A01020`） |
| sort_order | INTEGER | 表示順 |

#### products（商品テーブル）
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー、自動採番 |
| parent_number | TEXT | 親番号（例: `lk027`）、ユニーク |
| prefix_id | INTEGER | prefixesテーブルへの外部キー |
| created_at | DATETIME | 作成日時 |
| updated_at | DATETIME | 更新日時 |

#### designs（デザインテーブル）
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー、自動採番 |
| product_id | INTEGER | productsテーブルへの外部キー |
| design_letter | TEXT | デザイン番号（a～y） |
| image_url | TEXT | 画像のURL（Vercel Blob） |
| comment | TEXT | コメント（NULL許容） |
| created_at | DATETIME | 作成日時 |
| updated_at | DATETIME | 更新日時 |

**ユニーク制約**: `product_id` + `design_letter` の複合ユニーク

## 5. API設計

| メソッド | エンドポイント | 説明 |
|---|---|---|
| GET | `/api/search?q={検索文字列}` | 親番号の前方一致検索 |
| GET | `/api/products/{親番号}` | 商品詳細（デザイン一覧）取得 |
| POST | `/api/admin/upload/images` | 画像一括アップロード |
| POST | `/api/admin/upload/excel` | Excelアップロード |
| PUT | `/api/admin/designs/{id}` | デザイン情報更新 |
| DELETE | `/api/admin/designs/{id}` | デザイン削除 |
| DELETE | `/api/admin/products/{id}` | 商品一括削除 |
| GET | `/api/admin/products` | 管理用商品一覧取得 |
| GET | `/api/prefixes` | プレフィックス一覧取得 |
| POST | `/api/admin/prefixes` | プレフィックス追加 |

## 6. ファイル名バリデーション規則

```
正規表現: /^([a-zA-Z]+)(\d{3})-([a-y])\.(jpg|jpeg|png)$/i

グループ1: プレフィックス（登録済みプレフィックスと照合）
グループ2: 3桁の数字（001～999）
グループ3: デザイン番号（a～y）
グループ4: 拡張子
```

## 7. 画像処理

| 項目 | 仕様 |
|---|---|
| アップロード上限 | 1ファイルあたり5MB |
| 保存形式 | オリジナルのまま保存（JPG/PNG） |
| サムネイル | 表示時にNext.js Image Optimizationで自動リサイズ |
| ストレージ | Vercel Blob Storage |
