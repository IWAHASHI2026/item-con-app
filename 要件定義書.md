# 要件定義書: 商品デザイン確認アプリ

## 1. プロジェクト概要

### 1.1 目的
商品出荷時の商品デザイン確認を効率化するためのWebアプリケーション。
現場スタッフがスマホ・タブレットから商品管理番号で検索し、デザイン画像とコメントを素早く確認できるようにする。

### 1.2 利用者
- **現場スタッフ**（スマホ/タブレット）: 商品検索・デザイン確認
- **管理者**（PC）: 商品情報のアップロード・編集・削除

### 1.3 利用規模
- 同時アクセス数: 最大5名
- 認証・権限管理: 不要

---

## 2. 機能要件

### 2.1 商品検索機能（現場スタッフ向け）

| 項目 | 内容 |
|---|---|
| 検索対象 | 親番号（例: `lk027`） |
| 入力方法 | プレフィックスボタン + テンキー、またはキーボードによるフリー入力 |
| 検索方式 | 前方一致 |
| 1件ヒット時 | 直接、商品ページ（デザイン一覧）へ遷移 |
| 複数ヒット時 | 検索一覧画面（テキスト一覧）を表示 → タップで商品ページへ遷移 |
| 0件ヒット時 | 「該当する商品が見つかりません」メッセージを表示 |

### 2.2 商品ページ（デザイン表示）

| 項目 | 内容 |
|---|---|
| デザイン一覧 | 親番号に紐づくデザイン画像のサムネイル一覧を表示 |
| デザイン詳細 | サムネイルタップで拡大画像 + コメントを表示 |
| ズーム機能 | ピンチイン・ピンチアウトに対応 |

### 2.3 商品情報管理機能（管理者向け・PC操作）

#### 2.3.1 画像アップロード
| 項目 | 内容 |
|---|---|
| アップロード方式 | 一括アップロード（複数ファイル選択） |
| 対応フォーマット | JPG, PNG |
| 1画像あたりのサイズ上限 | 5MB |
| ファイル名規則 | `{親番号}-{デザイン番号}.{拡張子}`（例: `lk027-b.jpg`） |
| 自動判別 | ファイル名から親番号・デザイン番号を自動で判別し登録 |
| 重複時の挙動 | 同一管理番号の画像は上書き |

#### 2.3.2 コメントアップロード（Excel）
| 項目 | 内容 |
|---|---|
| フォーマット | A列: 管理番号（例: `lk027-a`）、B列: コメント |
| ファイル形式 | .xlsx |
| 挙動 | 既存コメントがある場合は上書き |

#### 2.3.3 編集・削除
| 項目 | 内容 |
|---|---|
| 編集 | 画像の差し替え、コメントの編集が可能 |
| 削除 | 個別デザインの削除、親番号単位での一括削除が可能 |

---

## 3. 非機能要件

| 項目 | 内容 |
|---|---|
| プラットフォーム | Webアプリ（ブラウザ） |
| 対応デバイス | スマホ・タブレット（検索・閲覧）、PC（管理画面） |
| ホスティング | Vercel |
| オフライン対応 | 不要 |
| 認証 | 不要 |
| バーコード/QR読み取り | 不要 |

---

## 4. データ仕様

### 4.1 商品管理番号の体系

```
管理番号全体: {プレフィックス}{3桁数字}-{デザイン番号}
例: lk027-a

親番号:       {プレフィックス}{3桁数字}
例: lk027

デザイン番号: a ～ y（アルファベット順、最大25個）
```

### 4.2 プレフィックス
- 現時点: `lk`, `UV`, `UVCHY`, `cp` の4種類
- 今後追加される可能性あり（管理画面またはDB設定で追加可能にする）

### 4.3 データ規模
- 親番号: 現在約150商品、最大999商品（001～999）
- 1親番号あたりのデザイン数: 10～20個（最大25個）
- 1管理番号あたり: 画像1枚 + コメント1件
